# Event Scanner — Status

## What this project is
A stock event scanner that finds upcoming catalysts (earnings, etc.), scores
setups 0-100, and recommends a trade strategy (STRADDLE, LONG CALL, SHARES,
DAY TRADE). For Julia's own trading research. Outputs a static HTML dashboard
auto-updated daily via GitHub Actions.

## Hard-won gotchas
- **yfinance fails on GitHub Actions runners** — `get_current_price()` and `get_option_quote()` return `None`/`"possibly delisted"` for every ticker when run from GitHub Actions, even though identical code works locally. Root cause believed to be Yahoo blocking GH-hosted runner IP ranges. yfinance 0.2.54 already bundles `curl_cffi` internally by default, so adding a custom curl_cffi session (PR #5) had **no effect** — don't retry that approach. See "Pending" below for the real fix direction.
- **`pnl_note` text vs `option_entry_date`** — `set_option_baseline()` sets `option_entry_date` once, but the human-readable `pnl_note` is separately regenerated by `update_option_pnl()` only on success. If that fetch fails, the old note text persists verbatim into future days — any "as of today" wording in a note must embed an actual date, never the word "today" (fixed in PR #4, applies to any future note text too).

## Pending / blocked
- **Switch price/option data source for CI** — Twelve Data API (`TWELVEDATA_KEY` in `scanner.py`, currently unused) likely needed for `get_current_price()` since Yahoo is blocked on GH Actions IPs. Twelve Data has no options-chain endpoint, so `get_option_quote()` (real option P&L) may need a separate data source or stay "unavailable" on CI. Not yet scheduled — needs design discussion.
- **Rotate `TWELVEDATA_KEY`** — hardcoded secret in `scanner.py` (public repo). Scheduled task `event-scanner-rotate-twelvedata-key` fires 2026-06-12 10:00 AM: removes hardcoded key, moves to GitHub secret `TWELVEDATA_API_KEY`, rotates the key value, updates `daily_scan.yml`.
